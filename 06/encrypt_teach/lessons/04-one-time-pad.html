Stop 4

One-time pad

XOR the message with a truly random key that is as long as the message and never reused. This is the summit: provably unbreakable. Every possible plaintext of the right length is exactly as consistent with the ciphertext, so the ciphertext reveals nothing. And it is almost useless — for one operational reason that, when broken, is catastrophic.

How it works

Perfect secrecy, one bit at a time

Encryption and decryption are the same operation: XOR with the key. Because the key is random, each ciphertext bit is a fair coin flip, independent of the plaintext.

Key (random bytes, hex)
Ciphertext = plaintext ⊕ key
Decrypt = ciphertext ⊕ key

A different key would decrypt the very same ciphertext into a different, equally sensible message. That is what “reveals nothing” means, and why it is provably secure.

How it breaks

The two-time pad: reuse the key once, lose everything

The word one-time is load-bearing. Suppose the same pad K encrypts two messages. Watch what happens when the attacker XORs the two ciphertexts together:

C1 = P1 ⊕ K
C2 = P2 ⊕ K
C1 ⊕ C2 = (P1 ⊕ K) ⊕ (P2 ⊕ K) = P1 ⊕ P2 ⊕ (K ⊕ K) = P1 ⊕ P2

The key XORs against itself and vanishes. The attacker is left holding P1 ⊕ P2 — two English messages laid over each other, and no key was ever needed.

C1 ⊕ C2 = P1 ⊕ P2 (key already gone, shown as printable bytes)

Crib-drag both messages out

Now peel them apart. Guess a common word (a crib) that appears in one message and slide it along P1 ⊕ P2. Where it lands on real text in one message, it reveals the other message underneath.

What it forced

What it forced

The one-time pad is perfect and impractical: the key must be truly random, as long as all the traffic combined, shared in advance, and destroyed after one use. Reuse a single byte and it collapses. The working compromise: keep a short key and stretch it into a long, random-looking keystream with a machine you can re-key safely. Building that machine — approximating randomness by design — is the job of the block cipher.
