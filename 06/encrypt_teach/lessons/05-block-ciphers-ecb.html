Stop 5

Block ciphers & ECB

The one-time pad needed randomness we could not afford. A block cipher like AES is a machine that manufactures randomness from a short key: a fixed, reversible scramble of a 16-byte block that behaves like a random permutation. Claude Shannon named the two ingredients — confusion and diffusion. Get them right and the block looks random. Then bolt it to your data the obvious way, and watch it fail anyway.

How it works

Diffusion: one input bit should move half the output

This is real AES-128 (via the browser's Web Crypto), encrypting a single 16-byte block. Flip one bit of the input and a good cipher repaints roughly half the output bits — the avalanche effect. That sensitivity is how a deterministic function imitates a random one.

Confusion (each output bit depends on the key in a tangled way) and diffusion (each output bit depends on many input bits) are engineered approximations of the pad's true randomness — cheap enough to use, strong enough to trust.

How it breaks

The ECB penguin

A block cipher only encrypts 16 bytes. To encrypt a whole image you must chop it into blocks and choose how to chain them. The simplest choice — ECB, Electronic Code Book — encrypts each block on its own. Nothing is wrong with the cipher or the key. The lesson. A strong primitive used naïvely is a weak system. The cipher did its job on every block; the mode — how blocks relate — is where the secret leaked.

What it forced

What it forced

To hide position, each block's encryption must depend on where it sits — on a counter or on the previous block, plus a unique starting value (the IV/nonce). That is what modes of operation (CBC, CTR, …) provide. But making ciphertext unpredictable is not the same as making it unforgeable — and the next stop shows an attacker rewriting your message without ever knowing the key.
